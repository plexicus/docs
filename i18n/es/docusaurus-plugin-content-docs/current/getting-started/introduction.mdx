# Bienvenido a la Documentación de Plexicus \{#getting-started_introduction-1}

Plexicus es la plataforma líder de protección de aplicaciones nativas de la nube (CNAPP) impulsada por IA, creada para asegurar todo tu ciclo de vida nativo de la nube, desde el código hasta la nube. Diseñada para entornos modernos de desarrollo y nube, Plexicus proporciona una plataforma unificada para identificar, priorizar y remediar riesgos en tus aplicaciones, infraestructura y cargas de trabajo con una eficiencia sin precedentes.

Nuestra plataforma automatiza aspectos críticos de la seguridad de las aplicaciones, ayudando a tus equipos a:

  - **Optimizar los flujos de trabajo de seguridad**: Automatizar la identificación, enriquecimiento y remediación de vulnerabilidades para reducir la intervención manual hasta en un 95%.
  - **Garantizar el cumplimiento**: Mantener la adhesión a los estándares de la industria como PCI DSS, SOC 2, NIST 800-53 y más.
  - **Integrarse sin problemas**: Trabajar sin esfuerzo con tus herramientas existentes como GitHub, GitLab, Bitbucket y otras plataformas CI/CD.

Al aprovechar algoritmos avanzados de IA, Plexicus ofrece información accionable para asegurar cada etapa del ciclo de vida del desarrollo de software, desde la creación del código hasta la implementación. Explora esta documentación para desbloquear todo el potencial de Plexicus y transformar la forma en que tu organización protege sus activos digitales.

# Flujo de Trabajo \{#getting-started_introduction-2}

1.  **Iniciando el Análisis del Repositorio**:  
    Un **Administrador**, **Ciberoperador** o **Desarrollador** comienza por analizar un nuevo repositorio.

2.  **Creación de Hallazgos**:  
    Una vez que el repositorio es analizado, la IA genera hallazgos y sugiere mejoras.

3.  **Edición y Revisión del Archivo**:

      - El **Administrador** o **Ciberoperador** revisa y edita el archivo que contiene las mejoras impulsadas por IA.
      - Envían el archivo editado al **Desarrollador**.

4.  **Revisión y Finalización por parte del Desarrollador**:

      - El **Desarrollador** analiza las mejoras sugeridas.
      - Pueden realizar ediciones adicionales o finalizar los cambios creando una solicitud de extracción para actualizar el archivo.

# Diferencias de Roles \{#getting-started_introduction-3}

| **Acción** | **Administrador** | **Ciberoperador** | **Desarrollador** |
| ------------------------- | --------- | ------------- | ------------- |
| Agregar Miembros al Equipo       | Sí       | No            | No            |
| Analizar Repositorios      | Sí       | **No** | Sí           |
| Ver Todos los Hallazgos         | Sí       | Sí           | No            |
| Enviar Hallazgos al Equipo de Desarrollo | Sí       | Sí           | **No** |
| Editar Archivo                 | Sí       | Sí           | Sí            |
| Crear Solicitud de Extracción         | Sí       | No            | Sí           |
| Solicitar Revisión            | No        | No            | Sí            |
