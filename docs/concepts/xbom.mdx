# XBOM \{#concepts_xbom-1\}

XBOM (Exploitable Bill of Materials) is the dedicated Plexicus view for software-supply-chain risk: which third-party components your applications depend on, which of those have known exploits, and how exposure traces back to specific repositories and findings.

XBOM is reachable from the main sidebar at **XBOM** (`app.plexicus.ai/xbom`).

:::warning Documentation in progress
This page is a placeholder. The XBOM feature is shipped and visible in the UI, but the full documentation — what each panel shows, how to read the supply-chain graph, how XBOM relates to SBOM exports, and the workflow for triaging exploitable components — is still being written.

## What we know so far \{#concepts_xbom-2\}

- XBOM is a first-class section of the Plexicus UI (`/xbom` route, `sidebar.xbom_title` i18n key).
- It surfaces dependency-level risk, distinct from per-finding triage in `/findings`.
- Data comes from the same enrichment pipeline as Findings — Trivy SCA, Syft, Grype, and BlackDuck (when configured) feed it.
- The "X" in XBOM stands for *exploitable* — XBOM filters the broader dependency inventory to components with known exploit paths (EPSS-scored, CVE-mapped).
