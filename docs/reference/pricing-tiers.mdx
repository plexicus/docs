# Pricing & Plans \{#reference_pricing-tiers-1\}

Plexicus pricing is **fetched live** from Stripe (or Microsoft Marketplace, if you onboarded that way). This page does not list specific plan names or prices because they change — query the live API or open the **Plan & Pricings** page in the UI for the current state.

:::tip Where to look
- **In the UI:** *Plan & Pricings* page (`app.plexicus.ai/plan-pricings`) shows the plans and limits that apply to your tenant.
- **From the API:** `GET /pricings` returns the same data programmatically.
- **In your account:** *Subscription Detail* (`app.plexicus.ai/payment`) shows your current plan + usage.
:::

## How plans are structured \{#reference_pricing-tiers-2\}

Each plan defines:

- **Limits** — repositories, cloud accounts, team members, API tokens.
- **Features** — scheduled scans, AI remediation, bulk remediation, deep-reasoning enrichment, SSO, Jira / ServiceNow integration, custom domain binding.
- **Support tier** — community / email / priority.

Free-tier clients have scheduled scans disabled (the analysis-scheduler explicitly skips them). Other limits are enforced at the API level — when you hit a cap, the response is `409` with a message naming the limit.

## Where the limits bite \{#reference_pricing-tiers-3\}

<AccordionGroup>
  <Accordion title="Repository limit hit" icon="material-symbols:fork-right-outline">
    Adding a new application is blocked. Existing repositories continue scanning. Either upgrade or remove unused repos.
  </Accordion>
  <Accordion title="Cloud account limit hit" icon="material-symbols:cloud-outline">
    Same — `POST /create_cloud_scan` returns `409`. Existing cloud apps unaffected.
  </Accordion>
  <Accordion title="Free tier — no scheduled scans" icon="material-symbols:schedule-outline">
    The `analysis-scheduler` service explicitly skips free-tier clients:

    > For each repo, check client subscription (skip free tier)

    Manual scan triggers still work. Webhook-triggered scans depend on tier.
  </Accordion>
  <Accordion title="API token limit hit" icon="material-symbols:key-outline">
    `POST /user/generate-api-token` returns `409`. Revoke an unused token first via [Generate API Token](/docs/recipes/generate-api-token).
  </Accordion>
  <Accordion title="Team member limit hit" icon="material-symbols:group-outline">
    `POST /team` returns `409`. Remove an inactive member or upgrade.
  </Accordion>
</AccordionGroup>

## Self-hosted \{#reference_pricing-tiers-5\}

Self-hosted deployments use a license key issued by Plexicus, not Stripe. The key is set via the Helm chart's `licenseKey` value or a Kubernetes Secret.

License keys encode:

- Expiry date.
- Maximum allowed clients (multi-tenant) or unlimited (single-tenant).
- Feature flags (SSO, AI provider override, etc.).

Renewal is handled via [engineering@plexicus.ai](mailto:engineering@plexicus.ai). Expired keys put the deployment into a degraded mode — existing scans continue; new scans are blocked.

## Microsoft Marketplace \{#reference_pricing-tiers-6\}

Plexicus is available via the Azure Marketplace. Billing flows through your Azure subscription instead of Stripe; the plan structure and limits mirror the Stripe SKUs. Activation status is shown at `app.plexicus.ai/microsoft-marketplace`.

## Reading your current plan \{#reference_pricing-tiers-7\}

```bash
curl -X GET "https://app.plexicus.ai/client" \
  -H "Authorization: Bearer $TOKEN"
```

Look for `subscription_tier` and `subscription_features`. The shape comes from your billing provider — Stripe returns `product.name` and `metadata`, Microsoft Marketplace returns `planId`.
