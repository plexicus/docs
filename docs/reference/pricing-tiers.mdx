# Pricing Tiers \{#reference_pricing-tiers-1\}

Plexicus has five tiers backed by Stripe (SaaS) or pre-paid licensing (self-hosted Enterprise). Tier choice is per-organization and changes via Organization → Billing.

:::tip Read the live `GET /pricings`
This page is the canonical *what each tier means*. The exact prices and SKU details live in `GET /pricings` (authenticated) — query that endpoint for current pricing data, since prices change.
:::

## Where the limits bite \{#reference_pricing-tiers-3\}

What happens when you hit each cap:

<AccordionGroup>
  <Accordion title="Repository limit hit" icon="material-symbols:fork-right-outline">
    Adding a new application is blocked with `409 Repository limit reached for tier {tier}`. Existing repositories continue scanning. Either upgrade or remove unused repos.
  </Accordion>
  <Accordion title="Cloud account limit hit" icon="material-symbols:cloud-outline">
    Same — `POST /create-cloud-scan` returns 409. Existing cloud apps unaffected.
  </Accordion>
  <Accordion title="Free tier — no scheduled scans" icon="material-symbols:schedule-outline">
    Free clients are explicitly skipped by `analysis-scheduler`:

    > For each repo, check client subscription (skip free tier)

    You can still trigger manual scans, but the daily/weekly cadence is off.
  </Accordion>
  <Accordion title="API token limit hit" icon="material-symbols:key-outline">
    `POST /user/generate-api-token` returns 409. Revoke an unused token first via [Generate API Token](/docs/recipes/generate-api-token).
  </Accordion>
  <Accordion title="Team member limit hit" icon="material-symbols:group-outline">
    `POST /team` returns 409. Remove an inactive member or upgrade.
  </Accordion>
  <Accordion title="Audit log retention exceeded" icon="material-symbols:history-outline">
    Older entries are deleted automatically (per `AUDIT_RETENTION_DAYS`). For longer retention on a lower tier, export the log periodically — see [Export Findings](/docs/recipes/export-findings).
  </Accordion>
</AccordionGroup>

## Trial conversion \{#reference_pricing-tiers-4\}

Trial = a time-limited Pro tier (30 days by default). At the end:

- If a payment method is on file → auto-converts to **Essentials** (not Pro). Upgrade explicitly to stay on Pro.
- If no payment method → reverts to Free. Excess data (repos beyond limit, extra team members) is preserved but **read-only** until you upgrade or remove the excess.

## Self-hosted licensing \{#reference_pricing-tiers-5\}

Self-hosted deployments use a license key issued by Plexicus, not Stripe. The key is set via the Helm chart's `licenseKey` value or a Kubernetes Secret.

License keys encode:

- Tier (always Enterprise for self-hosted)
- Expiry date
- Maximum allowed clients (multi-tenant self-hosted) or unlimited (single-tenant)
- Feature flags (SSO, AI provider override, etc.)

Renewal is handled via [engineering@plexicus.ai](mailto:engineering@plexicus.ai). Expired keys put the deployment into degraded mode (existing scans continue; new scans blocked).

## Microsoft Marketplace \{#reference_pricing-tiers-6\}

Plexicus is also available via the Microsoft Marketplace (Azure-only billing). Tier mapping is the same; billing flows through your Azure subscription instead of Stripe. SSO with Azure AD is included on the marketplace SKU regardless of tier.

## Reading your current tier \{#reference_pricing-tiers-7\}

```bash
curl -X GET "https://app.plexicus.ai/client" \
  -H "Authorization: Bearer $TOKEN"
```

Look for `subscription_tier`. The full tier definition (including limits as currently provisioned for your account) is in `subscription_features`.
