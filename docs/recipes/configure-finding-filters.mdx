# Configure Finding Filters \{#recipes_configure-finding-filters-1\}

Findings filters are first-class objects. You save a filter once, name it, and activate it later ‚ÄĒ across sessions, across users (if shared), and per Findings tab (App / SCM / Cloud).

## Save a filter \{#recipes_configure-finding-filters-2\}

<Steps>
  <Step title="Build the filter">
    On the Findings page, click **Filter**, choose dimensions (severity, status, repo, tool, ‚Ä¶), and click **Apply Filter**.
  </Step>
  <Step title="Click Save Filter">
    A dialog asks for a name (e.g. *Critical-untriaged*) and a description (optional). The active tab determines which endpoint the filter is saved against:

    - App tab ‚Üí `POST /findings-filter/default/save`
    - SCM tab ‚Üí `POST /findings-filter/scm/save`
    - Cloud tab ‚Üí `POST /findings-filter/cloud/save`
  </Step>
  <Step title="Confirm">
    Plexicus stores the filter under your user account. Saved filters appear in the dropdown next to **Filter**.
  </Step>
</Steps>

## Activate a saved filter \{#recipes_configure-finding-filters-3\}

Click the saved-filter dropdown, pick the filter. Plexicus calls one of:

- `POST /findings-filter/activate/{name}` (App)
- `POST /findings-filter/scm/activate/{name}` (SCM)
- `POST /findings-filter/cloud/activate/{name}` (Cloud)

The page reloads with the filter applied. Active filter persists across sessions until you change it.

## Update an existing filter \{#recipes_configure-finding-filters-4\}

To modify an active filter without creating a new one, change the filter dimensions and click **Update Filter** (instead of Save). Calls:

- `PUT /findings-filter/default/update-activate` (App)
- `PUT /findings-filter/scm/update-activate` (SCM)
- `PUT /findings-filter/cloud/update-activate` (Cloud)

This updates and re-activates atomically.

## Delete a filter \{#recipes_configure-finding-filters-5\}

Open the saved-filter dropdown, hover over a filter, click the trash icon. Calls `DELETE /findings-filter/{tab}/delete/{name}`.

Deleted filters cannot be restored ‚ÄĒ Plexicus does not soft-delete saved filters.

## Inspect what's currently active \{#recipes_configure-finding-filters-6\}

Plexicus exposes endpoints to read the active filter without changing it ‚ÄĒ useful when scripting against the API:

| Tab | Endpoint |
|---|---|
| App | `GET /findings-filter/default/selected` |
| SCM | `GET /findings-filter/scm/selected` |
| Cloud | `GET /findings-filter/cloud/selected` |

Each returns the saved-filter list plus the currently active filter object.

## Filter schema cheatsheet \{#recipes_configure-finding-filters-7\}

A saved filter is a JSON document with three top-level keys:

```json
{
  "pagination": { "page": 0, "pageSize": 25 },
  "repository_filter": { "ids": ["repo123", "repo456"] },
  "field_selection": ["severity", "title", "file", "tool", "status"]
}
```

Plus dimension-specific keys: `severity`, `status`, `tool`, `cwe`, `owasp`, `date_range`, `tags`, `false_positive`, `mitigated`. The full schema is in [Finding Filter Schema (Reference)](/docs/reference).
