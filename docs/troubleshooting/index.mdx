# Troubleshooting \{#troubleshooting_index-1\}

When Plexicus isn't behaving, find the closest match below and follow the linked playbook. Each page lists symptoms first, then the diagnostic procedure with the exact commands to run.

If your problem doesn't match any of these, open an issue at [github.com/plexicus/docs](https://github.com/plexicus/docs) with logs and we'll add it.

## Quick checks \{#troubleshooting_index-3\}

Before diving into a specific page, run these:

<Steps>
  <Step title="Plexicus is up?">
    SaaS: [status.plexicus.ai](https://status.plexicus.ai).
    Self-hosted: `kubectl get pods -n plexicus` — every pod should be `Running`.
  </Step>
  <Step title="Your auth works?">
    `curl -H "Authorization: Bearer $TOKEN" https://app.plexicus.ai/session` returns `200` with your user info.
    `401` → token issue. `403` → role issue. See [Error Codes](/docs/reference/error-codes).
  </Step>
  <Step title="The relevant integration is connected?">
    `curl -H "Authorization: Bearer $TOKEN" https://app.plexicus.ai/integrations/scm/check_validity` returns the connection status of every SCM.
  </Step>
</Steps>

If those three pass, the problem is *specific* — pick the right playbook below.

## Where to look first \{#troubleshooting_index-5\}

For self-hosted deployments, four log surfaces:

| Symptom | Inspect |
|---|---|
| API errors | `kubectl logs -n plexicus deploy/fastapi` |
| Scan stuck | `kubectl logs -n plexicus deploy/worker -f` + Temporal UI |
| Plexalyzer hung | `kubectl logs -n plexicus deploy/plexalyzer-code -f` |
| Remediation hung | `kubectl logs -n plexicus deploy/codex-remedium -f` |

For SaaS, your visibility is the audit log + status page; if those don't surface the issue, contact support.
